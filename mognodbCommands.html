MongoDB commands
Number Commands Description
1 cls clears console
2 show dbs Browses all the names of databases
3 show collections Browses all the names of the collections inside the database
4 use yourDBname It uses the specified database, it creates database if there is not one
5 db.createCollection('yourCollectionName') Creates collection in the database that's being used

example - db.createCollection('teachers', {capped:true,size:10000000,max:100},{autoIndexId:false})
Explanation: If "capped" set to true we have to specify size, here is 10 megabyte of space in bytes for teacher field. "max" is how many documents can we store limited by number and 'autoIndexId' will set index automatically if set to true although it has to be specified separately as given above

6 db.dropDatabase() Drops the database that's being used
7 db.yourCollectionName.insertOne()

Inserts one document
example - db.myCollection.insertOne({name:'myName'})

8 db.yourCollectionName.insertMany()

Inserts multiple documents. Takes an array.
example - db.myCollection.insertMany([{name:'myName'},{name:'myName2'},{name:'myName3'}])

9 db.yourCollection.find() Browses all the documents of the database that is being used.
It has 2 parameters. Those are query and projection. Here's how it works:
db.yourCollection.find({},{name:true,_id:false});
It'll return only names with the object id if "_id:false" is removed
10 db.yourCOllection.find().sort() Can sort the browsed data.

Example - db.yourCOllection.find().sort({name:-1})
In this example it'll look for name or any specified field in the documents and will return them in negative order(z-a).
If it is one then (A-Z)

11 db.yourCOllection.find().limit() Limits the returned documents number by the value provided.

example - db.yourCOllection.find().sort({name:-1}).limit(5)
NOTE: we can chain "sort()" and other methods too

12 db.yourcollection.updateOne(filter,update)

updates a document selected by a field. It takes 2 parameters. Filter and update
example: db.yourcollection.updateOne({name:'john'},{$set:{dateOfBirth:12-5-1235}})
db.yourcollection.updateOne(name:'john'},{$unset:{fulltime:}}) NOTE: filter to search it with field value and update is what to update with which property and value. If there is no field like that it'll add one

13 db.yourcollection.updateMany(filter,update) Same as updateOne but it updates multiple documents at once

example: db.yourcollection.updateOne({paymentDue:true},{$unset:{studentShip:true}})
db.stdInfo.updateMany({age:23},{$set:{courses:["Japanese","bangla","ongko","ingrejy","arby","farsi","urdu"]}})
db.stdInfo.updateMany({},{$unset:{courses:' '}}) This example is for deleting a field from every document if not wanted NOTE:First argument remains blank if nothing to search for

14 $set To set a field with value
15 $unset unset a field from a document or every document
16 $exists Checks if a field exists with boolean value

example - db.yourcollection.updateOne({age:$exists:{true}},{age:''})

17 db.yourcollection.deleteMany() same syntax as updateOne,findOne or many

example - db.stdInfo.deleteMany({gpa:{$exists:false}})
18 Comparison operators
$ne,$lte,$gte,$gt,$lt - not equals, less than, greater than...You get the point I.G. ðŸ˜‰ 		Find documents that doesn't match this criteria.

example - db.yourcollection.find({age:{$ne:21}})
db.yourcollection.find({age:{$gte:21, $lte:30}}) - sets range like 20 >=30

19 / $nin - in, not in This one takes an array and if the elements of the array matches it'll return the document, for $nin it will return the documents that doesn't match the elements. Iterating an array is simple with this one

example - db.yourCollectionName.find({name:{$in:["Faraz","Ahmed","Jane Smith"]}})
db.yourCollectionName.find({name:{$nin:["Faraz","Ahmed","Jane Smith"]}})

20 $and / $or
$and needs both of the condition to be true and $or needs one of them to be true to return documents 		example - db.stdInfo.find({$or:[{name:"Farah"},{age:{$lte:22}}]})
21 $nor
Given conditions has to be false, if one of them is false it'll return the document. Similarity with or is it needs only one of the criteria to be fulfilled. $nor an or is here

example - db.stdInfo.find({$nor:[{courses:'english',courses:'physics'}]})

22 count() As the name suggests....it returns count of returned documents

example - db.stdInfo.find({$nor:[{courses:'english',courses:'physics'}]}).count()

23 $not If the given condition doesn't match it'll return the document

example - db.stdInfo.find({courses:{$not:{age:{lte:25}}}})
24 indexes An index allows a quick look up of a field, it slows down crud operations,
it takes up more memory,so have to use wisely
It stores data as b-tree.
If search is mostly used and not updates then it causes less hassle.
It is helpful for thousands of data. Because linear search will take a lot of time for big amount of data
25 db.collectionName.createIndex() example - db.collectionName.createIndex({name:1}) - ascending order, descending would be -1
26 db.yourcollection.getIndexes() to get the collections by index if created with index
27 db.yourcollection.explain() The explain command provides information on the execution of the following commands

example - db.yourcollection.explain('executionStats')
27 db.yourcollection.DropIndex() Deletes a document
